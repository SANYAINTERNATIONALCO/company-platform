# CLAUDE.md — منصة شركة سانيا الدولية (Company Platform)

## نظرة عامة
منصة إدارة داخلية لشركة سانيا الدولية (موارد بشرية + مالية) — مشروع سمنت الأمير.
واجهة عربية RTL بالكامل. المطوّر والمالك: حسن (مدير الموارد البشرية، دوره في النظام: editor).

## التقنيات
- Next.js 16 (App Router) + TypeScript — تطبيق صفحة واحدة client-side
- Supabase: قاعدة بيانات Postgres + Auth + Storage (مفتاح anon مكتوب مباشرة داخل كل مكوّن — لا تغيّره)
- النشر: Vercel تلقائياً عند `git push` إلى GitHub (SANYAINTERNATIONALCO/company-platform)
- تطبيق أندرويد: Capacitor يفتح الموقع المباشر (مجلد android/)
- مكتبات: recharts (رسوم بيانية)، xlsx (تصدير Excel)

## بنية المشروع
- `app/page.tsx` — تسجيل الدخول، الشريط الجانبي (مجموعات: الموارد البشرية/المالية/الإدارة/النظام)، لوحة المعلومات، تحميل الدور، عرض الأقسام
- `app/components/` — 13 قسماً مستقلاً: Employees, Attendance, Payroll, Documents, Custody, Contracts, Overtime, Finance, Receipts, Visa (يشمل دورات المغادرة والعودة), Tasks, Reports, ActivityLog
- `app/logActivity.ts` — دالة مشتركة لسجل النشاطات
- كل مكوّن ينشئ عميل supabase خاصاً به، وستايلات inline (لا يوجد CSS framework)

## قواعد حرجة — لا تخالفها أبداً
1. استخدم `React.ReactElement` وليس `JSX.Element` (البناء يفشل معه)
2. `npm run build` يجب أن ينجح قبل أي `git push`
3. حالات الحضور نصوص عربية حرفية في قاعدة البيانات — لا تغيّرها إطلاقاً:
   `'حاضر'، 'غائب'، 'روتيشن'، 'يوم جمعة'، 'إجازة اعتيادية'، 'إجازة مرضية'، 'إجازة طارئة'، 'إجازة تعويضية'، 'إجازة وفاة'، 'عطلة رسمية'`
   (الحالة القديمة `'إجازة'` تُحتسب اعتيادية)
4. سجّل العمليات الحساسة عبر `logActivity(action, section, details)`: كل حذف، تعديل مالي، أرشفة/حذف رواتب، توقيعات، إصدار/حذف كتب، تسجيل دخول
5. أعمدة view العرض الشهري `monthly_attendance_summary` بأسماء عربية — طابقها حرفياً
6. كل واجهة جديدة: عربية RTL بنفس نمط البطاقات/الألوان الموجود

## الأدوار والصلاحيات
- `editor` (حسن): كل شيء
- `admin` (مدير الموقع): قراءة فقط + رفع توقيعه فقط في الرواتب + رؤية كل المهام + طباعة الكتب المحفوظة
- `accountant`: الحسابات + المهام فقط
- `guest_1` / `guest_2`: قراءة فقط لكل شيء
- النمط: `isReadOnly = ['admin','guest_1','guest_2'].includes(userRole)` في page.tsx، وتمرير `readOnly` prop للمكونات (استثناء: Payroll يستلم `userRole` أيضاً، وTasks يستلم بيانات المستخدم)

## قواعد العمل (Business Rules)
- الرواتب: يُخصم الغياب فقط (اليوم = الراتب÷30)، كل الإجازات مدفوعة
- الإجازة الاعتيادية: يومان شهرياً غير تراكمية (استخدمها أو اخسرها) لموظفي الدوام اليومي؛ الروتيشن: أسبوع شهرياً لا يُخصم؛ الوفاة: 5 أيام مدفوعة؛ المرضية: بلا حد
- أرشفة الرواتب: `payroll_records` (فريد لكل موظف+شهر)، عداد السلفة يتحدث تلقائياً عند أول حفظ ويرتد عند حذف الأرشفة؛ الموافقات الشهرية في `payroll_approvals` (توقيع مستقل لكل شهر)
- دورات الفيزا: فترة سماح 60 يوماً بعد انتهاء الفيزا → فيزا مغادرة 10 أيام (تحذير ≤5) → مغادرة → عودة (اكتمال)؛ `group_name` للإجراءات الجماعية؛ الحالات: grace_period / exit_visa_issued / departed / completed
- الكتب الرسمية: ترقيم `HR-{TYPE}-{YEAR}-{NNN}` عبر `document_counters`؛ المحتوى الكامل في `doc_content` JSONB لإعادة الطباعة؛ الترويسة/التذييل صور في `document_assets`
- الأوفرتايم: قسم Overtime.tsx يسجل أيام الأوفرتايم بجدول `overtime_records`، كل يوم = +1 لعمود `employees.overtime_leave_balance` (رصيد تراكمي منفصل تماماً عن الإجازة الاعتيادية). يُستهلك الرصيد (−1) عند تسجيل يوم حضور بحالة `'إجازة تعويضية'` بـAttendance.tsx، ويرتد (+1) عند تغيير/حذف ذلك اليوم. يسمح بتجاوز الرصيد (تحذير فقط دون منع، نفس نمط الإجازة الاعتيادية)
